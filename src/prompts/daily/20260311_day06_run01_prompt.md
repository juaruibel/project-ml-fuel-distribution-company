# Day 06 · Prompt operativo (run01)

## Objetivo
Cerrar el salto de modelo a producto sin reabrir líneas de ML ya cerradas, separando dos superficies dentro de una única app Streamlit:

1. `Demo`: narrativa, comparación de referencias y evidencia resumida.
2. `Producto`: input realista, inferencia operativa, feedback humano y persistencia auditable.

## Restricciones activas
- No abrir Brent ni forecast.
- No reentrenar modelos.
- No promocionar nuevas policies Day 05.4x.
- No cambiar el serving default histórico sin decisión explícita de rollout.
- Mantener SQL como espejo opcional y no como requisito para inferencia local.

## Referencias activas del producto
- `baseline`: `LR_smote_0.5` como serving default histórico congelado.
- `champion_pure`: `V2_TRANSPORT_ONLY_LIGHTGBM_CLASS_WEIGHT_BALANCED_v1` como default recomendado para rollout conservador.
- `baseline_with_policy`: `V2_TRANSPORT_ONLY_LR_smote_0.5_WITH_DETERMINISTIC_LAYER_PRODUCT_002_FOLLOW_PRODUCT_003_SUPPLIER_009_v1` como alternativa operativa visible.

## Contrato de Day 06
- `app.py` queda como shell fino con selector `Demo` / `Producto`.
- La lógica reusable sale a `src/ml/ui/*` y `src/ml/product/*`.
- La persistencia operativa canónica queda en:
  - `data/raw/inference_inputs/<YYYYMMDD>/`
  - `data/processed/inference_inputs/<YYYYMMDD>/`
  - `artifacts/public/inference_runs/<YYYYMMDD>/`
  - `artifacts/public/inference_feedback/<YYYYMMDD>/`
- Se conserva compatibilidad con la trazabilidad histórica en `artifacts/public/postinferencia/audits/<YYYYMMDD>/`.
